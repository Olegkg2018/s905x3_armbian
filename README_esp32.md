# 🔋 ESP32 JK-BMS & Sumry Inverter Gateway

Универсальный шлюз на базе **ESP32 (ESP32-ETH01)** для мониторинга и управления системой накопления энергии.
Объединяет данные от BMS (по Bluetooth) и Гибридного инвертора (по RS485) в единую экосистему Home Assistant через ESPHome.

## 🌟 Возможности
- **JK-BMS**: Чтение напряжений ячеек, токов, температур, SOC, управление балансировкой и реле (через BLE).
- **Sumry CP3200**: Опрос данных инвертора (напряжение сети, батареи, мощность, ошибки) через Modbus RTU.
- **Интеграция**: Полная нативная поддержка в Home Assistant.
- **Аппаратная часть**: Использование изолированного преобразователя TTL-RS485 для надежной связи.

## 🔌 Схема подключения (Wiring Diagram)

Используется плата **ESP32-ETH01** (WROOM/WROVER) и модуль **Isolated TTL to RS485**.

### Pinout ESP32 -> RS485 Module
| ESP32 GPIO | RS485 Module | Описание |
| :--- | :--- | :--- |
| **GPIO 16** | `DI` (Data In) | TX (UART2) |
| **GPIO 17** | `RO` (Data Out) | RX (UART2) |
| **GPIO 4** | `DE` & `RE` | Direction Control (связаны вместе) |
| **3.3V** | `VCC` | Питание модуля |
| **GND** | `GND` | Общая земля |

### RS485 Module -> Sumry CP3200 Inverter
| RS485 Module | Inverter Port (RJ45/Terminal) | Описание |
| :--- | :--- | :--- |
| **A** (D+) | `485+` / `D+` | Линия A |
| **B** (D-) | `485-` / `D-` | Линия B |
| **GND** | `GND` | **Важно:** Соединить земли! |

> **Примечание:** JK-BMS подключается беспроводным способом. Убедитесь, что MAC-адрес в конфиге совпадает с вашим.

## ⚙️ Установка
1. Установите аддон **ESPHome** в Home Assistant.
2. Создайте новое устройство, выберите "Manual Edit".
3. Скопируйте содержимое файла `jk-sumry-inverter.yaml` из этого репозитория.
4. Отредактируйте секцию `substitutions`:
   - Замените `mac_address` на адрес вашего JK-BMS.
   - При необходимости скорректируйте `ssid` и `password`.
5. Нажмите **Install** и прошейте устройство.

## 🛠 Настройка Modbus для инвертора
В файле конфигурации используются типовые адреса регистров для инверторов типа Solis/Sumry.
Если данные не читаются:
1. Включите логгер (`level: DEBUG`).
2. Проверьте адреса регистров в инструкции к вашему инвертору.
3. Отредактируйте значения `address` в секции `sensor: platform: modbus_controller`.
