# 🎙️ EchoKit-сервер «Алиса» на S905x3 (альтернатива xiaozhi-esp32-server)

Прототип второго варианта серверной части умной колонки — на движке
[second-state/echokit_server](https://github.com/second-state/echokit_server)
(Rust) вместо xiaozhi-esp32-server (Python). ASR/LLM/TTS подключаются как
произвольные OpenAI-совместимые API, без привязки к конкретному облаку.

Полный пайплайн (ASR-эхо → LLM → TTS → аудио-стриминг по протоколу сервера)
проверен end-to-end локально: Groq ASR распознал сгенерированную Supertonic
русскую фразу дословно верно, DeepSeek/Groq LLM ответил по-русски в
характере системного промпта, первый аудио-чанк ответа пришёл через ~0.3с
после отправки текста. **Сборка образа на ARM не тестировалась** — в
песочнице, где готовился этот прототип, нет Docker/root — нужно проверить
на самом S905x3.

## Что здесь

* `Dockerfile.server` — сборка `echokit_server` из исходников (git-клон
  апстрима, пин на конкретный коммит для воспроизводимости), нативно под
  архитектуру хоста — на S905x3 (aarch64) даст aarch64-бинарник без
  кросс-компиляции.
* `Dockerfile.supertonic` — локальный TTS-сервер
  [Supertonic](https://github.com/supertone-inc/supertonic) с OpenAI-совместимым
  эндпоинтом `/v1/audio/speech`, работает на CPU, без GPU. Поддерживает
  русский (31 язык всего).
* `docker-compose.yml` — поднимает оба сервиса вместе, `restart: unless-stopped`
  для автозапуска после перезагрузки/падения (Docker должен быть включён как
  systemd-сервис — это по умолчанию так на Armbian).
* `config/config.toml` — конфиг: Groq ASR, DeepSeek LLM, локальный Supertonic
  TTS, системный промпт Алисы на русском. Ключи нужно вписать перед запуском.
* `config/hello.wav` — приветственная фраза, которую сервер шлёт устройству
  при подключении (сгенерирована тем же Supertonic).

## Перед запуском

Впишите ваши реальные ключи в `config/config.toml`:
```toml
[asr]
api_key = "ВАШ_GROQ_API_KEY"
...
[llm]
api_key = "ВАШ_DEEPSEEK_API_KEY"
```
⚠️ На момент подготовки этого прототипа на DeepSeek-ключе, использованном для
теста, был нулевой баланс (`Insufficient Balance`) — сама аутентификация
прошла, ключ рабочий, но нужно пополнить счёт. Пока баланс не пополнен,
можно временно указать в `[llm]` URL и ключ Groq
(`https://api.groq.com/openai/v1/chat/completions`, модель например
`llama-3.3-70b-versatile`) — тот же ключ, что уже используется для ASR.

## Запуск

```bash
cd echokit-server
docker-compose up -d --build
```

Первая сборка `echokit_server` — это полноценная Rust-компиляция с нуля
(зависимости: burn/onnx для встроенного локального VAD, opus, MCP SDK и
т.д.) — на слабом ARM SBC может занять заметно дольше, чем на десктопе.
Это одноразовые затраты, дальнейшие правки конфига пересборки не требуют.

Supertonic при первом запросе скачает модель с HuggingFace (~сотни МБ) —
это тоже одноразово, кешируется в volume `supertonic-cache`.

## Проверка

```bash
docker-compose logs -f echokit-server
```

Сервер слушает WebSocket на `:8080/ws/{id}`. Полноценно проверить голосом
можно, только подключив реальное ESP32-устройство с портированной прошивкой
(следующий шаг — портирование `echokit_box` под вашу плату N16R8, ещё не
сделано). Проверить сам сервер и связку ASR/LLM/TTS без микрофона можно
текстовой командой по протоколу — подключиться к `ws://<IP>:8080/ws/test` и
отправить текстовый фрейм `{"event":"Text","input":"Привет!"}` после
получения `HelloEnd` — сервер прогонит это через LLM и TTS так же, как
реальную распознанную фразу.
